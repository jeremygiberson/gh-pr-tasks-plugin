## Source Code Findings

### Command Execution ‚Äî SAFE

All external commands flow through `internal/ghcli/ghcli.go` via `exec.Command("gh", args...)`. This uses Go's argument array form which does not invoke a shell. User-supplied values are passed as:
- Individual `-f key=value` arguments to the `gh` CLI
- JSON-serialized GraphQL payloads piped via stdin (`--input -`)

No use of `sh -c`, `os.system`, or shell string interpolation anywhere in the codebase.

### Network Communication ‚Äî SAFE

All network activity is delegated to the `gh` CLI binary (`gh api` and `gh api graphql`). There are:
- No `net/http` imports or direct HTTP clients
- No telemetry, analytics, or phoning-home behavior
- No connections to any service other than the configured GitHub host
- The only domain literal is `"github.com"` as a default fallback in `resolver.go:sanitizeHost()`

### Secrets and Obfuscation ‚Äî CLEAN

- No hardcoded tokens, API keys, or passwords
- No base64 encoding/decoding of payloads
- No obfuscated strings, hex-encoded data, or runtime code generation
- Authentication is entirely delegated to `gh` CLI's own auth mechanism

### Hidden Functionality ‚Äî NONE FOUND

- Command tree is fully visible in `cmd/root.go`: `comments reply`, `review` (start/add-comment/submit), `review view`, `threads` (list/resolve/unresolve)
- No hidden subcommands, undocumented flags, or environment variable side channels (beyond standard `GH_HOST`)
- No `init()` functions with hidden behavior, no background goroutines
- All GraphQL mutations map to standard GitHub PR review API operations

### Input Validation ‚Äî ADEQUATE

| Input | Validation |
|---|---|
| PR selector (number/URL) | Regex + `strconv.Atoi` in `resolver.go` |
| `--repo` flag | `splitRepo()` validates `owner/repo` format |
| Review IDs | Must start with `PRR_` prefix |
| `--side` | Restricted to `LEFT` or `RIGHT` |
| `--event` | Restricted to `APPROVE`, `COMMENT`, `REQUEST_CHANGES` |
| `GH_HOST` | `sanitizeHost()` strips protocols, ports, paths, lowercases |
| `--body` | Trimmed, passed as-is (correct for user-authored content) |

### Dependencies ‚Äî MINIMAL AND CLEAN

Direct:
- `github.com/spf13/cobra v1.9.1` ‚Äî widely-used CLI framework
- `github.com/stretchr/testify v1.9.0` ‚Äî standard testing library

All transitive dependencies are well-known Go ecosystem packages: `go-spew`, `mousetrap`, `go-difflib`, `pflag`, `yaml.v3`.

## CI/CD and Supply Chain Findings

| Finding | Risk | Detail |
|---|---|---|
| Unpinned GitHub Actions | MEDIUM | `actions/checkout@v4`, `actions/setup-go@v5`, `golangci/golangci-lint-action@v6`, `cli/gh-extension-precompile@v2` use mutable tags instead of commit SHAs |
| golangci-lint `version: latest` | LOW-MEDIUM | Lint tool version not pinned; downloads latest binary at CI runtime |
| No binary checksums/signatures | MEDIUM | Release binaries (12 platforms) have no published checksums or SLSA provenance |
| `workflow_dispatch` on release | LOW-MEDIUM | Anyone with write access can trigger a release build manually |
| Shell interpolation in release.yml | LOW | `TAG="${{ github.ref_name }}"` is a known anti-pattern (git tag name restrictions limit practical exploitation) |
| CODEOWNERS org mismatch | LOW | `CODEOWNERS` references `@hautechai/humans` but repo is owned by `agynio` ‚Äî relationship unclear |
| Branch protection unknown | MEDIUM | API returned null for branch protection rules on `main` |

## Conclusion

**The source code is safe to use.** No backdoors, data exfiltration, command injection, or malicious behavior were found. The codebase is well-structured with a single point of external command execution (`internal/ghcli/ghcli.go`) making it straightforward to audit.

The CI/CD concerns are typical of small open-source projects and do not indicate malicious intent. They represent standard supply chain hygiene improvements that could be made. Since we install from source via `--pin <commit>`, the binary supply chain concerns are mitigated.
